Autolamella Test Checklist

Checklist for functionality of program

This checklist is to be used from a user perspective to test for functionality and test for aspects which cannot otherwise be automatically tested. It is also used to test for functionality of the program as a whole, and not just the individual components. It also provides a sanity check and outlines expected behaviour of the program.

This checklist is not exhaustive and is subject to change.

Connection

Connecting to microscope

  1. Does not crash program if connection is unsuccessful,
  2. Able to retry connection with different settings if required
  3. Can disconnect successfully
  4. Can reconnect successfully

Experiment Creation

  1. Can create experiment
  2. Can load a previously created experiment
  3. 2.1 Successfully loads past lamella at correct stage and positions

Protocol Loading

  1. Can load a protocol
  2. Can load a protocol without errors
  3. Loads default values for keys that are not provided
  4. Key type is asserted
  5. Can save a protocol to file
  6. protocol is saved correctly, modified values are saved accurately
  7. Value types are maintained
  8. Saved protocol can be reloaded successfully
  9. Can save to a new or existing protocol file
  10. Can modify a protocol live and resulting pattern updates live

Adding/Moving/Removing Lamella and Fiducial patterns

  1. Can add lamella pattern
  2. Can add empty lamella
  3. Can alter lamella parameters
  4. Can move lamella pattern to a new position
  5. Can move by specifying x and y coordinates
  6. Can move by clicking on image
  7. Moving lamella does not alter any parameters other than position
  8. Moving lamella does not alter fiducial pattern
  9. Moving lamella/fiducial does not alter other lamella/fiducial patterns
  10. Results in warning when clicking out of bounds
  11. Cannot move lamella outside of image.
  12. Can remove lamella
  13. Removes correct lamella
  14. In experiment list, each lamella has a unique name
  15. Lamella is removed from list of available lamellae
  16. Can add/remove microexpansions

Saving Lamella and Milling Fiducial

  1. Can save lamella
  2. Warning shown if Horizontal Field Width (HFW) is too big or small for pattern
  3. Saving Lamella mills fiducial pattern
  4. Can remill fiducial
  5. Can move to position once lamella is saved, allowing user to move to another location for another lamella
  6. If lamella is not saved, position is not saved, move to position button is disabled

Running Autolamella

  1. Can run autolamella
  2. Lamellae are milled accurately and in position (check physically)
  3. Lamellae are milled in order of creation
  4. Can run with/without microexpansions
  5. Lamella data only saved if stage completed successfully

Consistency across manufacturers

  1. UI and behaviour is consistent across manufacturers where expected
  2. Certain features not available and is hidden away depending on manufacturer (e.g. Presets hidden for ThermoFisher system, application file hidden for TESCAN system)

Lamella Status

  1. Lamellae have status set and updated throughout each step

  2. Steps

    • Setup
    • Fiducial Milled
    • Microexpansion cut
    • Rough cut
    • Regular cut
    • Polishing cut
    • Finished

  3. Status can be accessed anytime and saved to experiment
